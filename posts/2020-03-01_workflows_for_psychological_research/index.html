Workflows for psychological research

2020-03-01

I’ve been thinking about workflows for writing scientific documents (e.g., research papers, essays, and so on). It seems to me that the sanest way to go about these things would be to do one’s writing in RMarkdown documents, with external .bibtex files storing the references (and git handling version control). Everything gets processed in knitr and pandoc, and out comes a PDF file and a .docx file formatted according to APA-standards. The entire project (containing the raw data, plots, etc.) could be hosted on GitHub, OSF, or on some other service. This would keep things tidy, transparent, and reproducible. Importantly, it would allow one to focus on the writing, and not on wrestling with the formatting.

The trouble is, of course, that a workflow like this takes a while to set up in order to get it right. The setup also requires quite a bit of familiarity with the underlying technologies, making it inaccessible for sane people who do not spend their free time reading technical documentation.

Another option, of course, would be to just go back to using typewriters, and not worry about compilation errors or file encodings. I’m half-joking. I think.
